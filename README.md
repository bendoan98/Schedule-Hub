# Schedule Hub

Schedule Hub is a React + Supabase app for team scheduling with:
- weekly calendar views
- manager tools for importing and maintaining schedules
- shift/schedule request workflow
- team-based onboarding with invite codes
- individualized notifications with per-user read state

## Current Features

### Auth and Team Onboarding
- Email/password auth via Supabase Auth.
- Sign-up requires only `Full Name`, `Email`, and `Password`.
- After first sign-in, users must:
  - create a team (becomes manager), or
  - join an existing team with invite code.
- `employees` rows are auto-created from Auth users via DB trigger.

### Role-Based Experience
- **Managers**
  - Can open **Manager Page** for operations.
  - Can edit shifts from the weekly calendar.
  - Can manage departments and update team member departments (including their own).
- **Employees**
  - See schedule rows for themselves and teammates in the same department.
  - Can request swaps only on their own shifts.
  - Cannot edit shifts.

### Manager Page
- CSV import with a selectable target week (`Week of` date).
- Schedule request queue (approve/deny).
- Department Manager panel:
  - add / rename / delete departments
- Team Roster panel:
  - assign department from available department list.

### Schedule Page
- Weekly calendar with Monday-Sunday grid.
- Prev/Next week navigation.
- Export dropdown next to **Weekly Calendar** title:
  - Export Personal (`.ics`)
  - Export Team (`.ics`, manager only)
  - Open next shift in Google Calendar
- Message board is shown on schedule page (not manager page).
- Dashboard stats are shown on schedule page only.

### Notifications
- Notification bell with unread count.
- Click outside the popover closes it.
- "Mark all read" marks only the current user's notifications.
- Notifications are individualized by recipient and read state is per-user.
- Sender is excluded from notifications generated by their own action.

Notification triggers in current implementation:
- CSV import -> employees receive `New Schedule Available`.
- Department update/rename/delete -> affected employees only.
- Schedule request submission -> managers only.
- Schedule request approval/denial -> addressed employee(s) only.

## Supabase Data Model (Current)

Core tables:
- `teams`
  - includes `invite_code`, `created_by`
- `departments`
  - `team_id`, `name` (unique per team)
- `employees`
  - `id` (matches `auth.users.id`), `team_id`, `role`, `department_id`, `color_index`
- `shifts`
  - `employee_id`, `day`, `start_time`, `end_time`, `week_start`
- `swap_requests`
  - `shift_id`, `requested_by`, `status` (`pending|approved|denied`)
- `notifications`
  - `recipient_employee_id`, `sender_employee_id`, `read_at`
  - supports per-recipient inbox and per-user read state
- `message_posts`
  - team message board posts

DB functions/triggers:
- `handle_new_user` trigger -> creates `employees` profile on auth sign-up
- `create_team_for_current_user(text)` RPC
- `join_team_with_invite_code(text)` RPC

## RLS Summary

High-level policy behavior:
- Team-scoped access by `current_team_id()`.
- Managers can manage team records (employees/departments/shifts/requests).
- Employees can:
  - read own profile
  - read same-team coworkers in same department
  - read shifts for self and same-department coworkers
  - insert swap requests for their own shifts only
- Notifications:
  - each user can read/update only their own notification rows (`recipient_employee_id = auth.uid()`).
  - inserts are allowed to team recipients according to role rules.
- Department linkage:
  - `employees.department_id` references `departments.id` with `ON DELETE SET NULL`.

See:
- `supabase/schema.sql`
- `supabase/rls.sql`

## CSV Format

Accepted columns:
- `role`
- `employee_name`
- `monday` `tuesday` `wednesday` `thursday` `friday` `saturday` `sunday`

Example:

```csv
role,employee_name,monday,tuesday,wednesday,thursday,friday,saturday,sunday
employee,Jamie Chen,09:00-17:00,09:00-17:00,,13:00-21:00,,09:00-17:00,
manager,Alex Morgan,,,,,,,
employee,Sam Rivera,,07:00-15:00,,07:00-15:00,,15:00-23:00,
```

Notes:
- Time format: `HH:MM-HH:MM`.
- Import uses the selected **Week of** date and normalizes to that week's Monday.
- In Supabase mode, shifts are imported only for employees already in the team roster (unknown names are skipped).

## Local Development

### Prerequisites
- Node.js 18+
- npm
- Supabase project (optional for mock mode)

### Install

```bash
npm install
```

### Environment

```bash
cp .env.example .env.local
```

Set:
- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_ANON_KEY`

If env vars are missing, app runs in local mock-data mode.

### Run

```bash
npm run dev
```

Build:

```bash
npm run build
```

Preview production build:

```bash
npm run preview
```

## Supabase Setup

Run SQL in this order in Supabase SQL Editor:
1. `supabase/schema.sql`
2. `supabase/rls.sql`
3. `supabase/seed.sql` (optional)

Then:
1. Start app and sign up with name/email/password.
2. Create a team or join a team with invite code.
3. Managers can switch between **Schedule** and **Manager Page** from the header.
